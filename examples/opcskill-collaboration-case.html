OPCSkill Collaboration Case

AI 的回答可以重来，人的判断不能丢。

多个 skill 协作，把一段 AI 对话转成可理解、可复用、可视化的公司资产：先理解，再编译，再沉淀，最后用 HTML 作为人的审查和思考工作台。

从原始对话到下一轮 AI 资产

offline / single file

这个 artifact 不是生产清洗工具，而是演示 OPCSkill 如何协调技能，把一个有价值的 AI 对话变成下一轮可用的资产。

处理前 / 处理后

同一段对话，未处理时只是聊天记录；经过 OPCSkill 协调后，它变成人可以审查、确认、复用的知识资产。

Before：散落的对话

  • 对话散落在窗口里，下一次很难精确找回。
  • 人的决策和模型建议混在一起，权威来源不清。
  • Prompt 和任务包无法复用，只能靠记忆重写。
  • 下一轮 AI 需要重新解释背景，成本被反复支付。

After：可复用的资产

  • 人的决策被标记和保留，进入 Decision Ledger。
  • 概念结构被解释清楚，假设、机制、边界分离。
  • Prompt / Task Packet 被提取，下一轮可以直接使用。
  • Markdown 成为长期真源，HTML 成为可视化审查和理解工作台。

Skill 协作地图

OPCSkill 的价值不只是“保存 Markdown”，而是把已有技能编排成一个可解释的协作链。

lijie

理解结构 / concept map / first-principles decomposition

  • 提取核心问题
  • 建立概念结构
  • 分离假设、机制和约束

token-prompt-compiler

把对话编译成可交给下一轮 AI 的任务资产。

  • 生成可复用提示卡
  • 生成执行任务包
  • 加入验证器和停止规则

evaluation

把可复用教训、决策记录和不确定性写入 Markdown。

  • 写出 Markdown 资产报告
  • 保留可复用经验
  • 分离已验证事实、推断和不确定性

claude-code-html-skill

把密集 Markdown 变成视觉工作台，让人审查、比较、导出。

  • 展示对照、证据、流程和导出
  • 帮助人审查和思考
  • 把确认后的内容导回 Markdown

Markdown + HTML Relationship

HTML 是工作台，不是长期记忆本身。RAG、eval 或 training 应读取确认后的干净资产，而不是原始聊天记录。

Markdown 是长期真源

保存已确认的人类决策、可复用 prompt、任务包、评估笔记和 open questions。它适合版本管理、搜索、后续 agent 读取。

已确认决策 资产报告 下一轮上下文

HTML 是可视化审查、思考和导出界面

把密集 Markdown 变成对照、流程、证据卡片和轻量导出。人的判断在这里被看见、过滤、复制回 Markdown。
